# Aviato requirements traceability

This is the canonical state and evidence ledger for numbered requirements,
threats, and security controls. Use the repo-local `traceability` skill when
changing it. The allowed states are `proposed`, `accepted`, `implemented`,
`verified`, `blocked`, and `retired`.

A `verified` row means the linked verification proves the repository-controlled
behavior represented by that row. Notes identify external operator gates that
remain outstanding; those are never implied by a local test.
